# LexoWords

LexoWords - Android-приложение для изучения английских слов с ежедневным лимитом, повторением по интервалам и локальным словарем. Проект сделан как дипломная работа и демонстрирует полный цикл работы с данными: загрузку словаря из assets, хранение прогресса в локальной базе, обучение, повторение и отображение статистики.

## Возможности

- изучение новых слов с дневным лимитом;
- карточки слов с переводом, транскрипцией и озвучиванием через Text-to-Speech;
- повторение слов по расписанию с расчетом следующей даты повторения;
- разделение слов по состояниям: новые, изучаемые, выученные и готовые к повторению;
- список слов с вкладками "Учу" и "Знаю";
- экран статистики со стриком, дневным прогрессом, количеством повторений и состояниями слов;
- автоматическая подготовка слов к повторению через WorkManager;
- локальное хранение словаря, тегов и профиля пользователя.

## Архитектура

Проект разделен на несколько слоев:

- `ui` - Compose-экраны, ViewModel и переиспользуемые UI-компоненты;
- `domain` - модели, репозитории и use case'ы бизнес-логики;
- `data` - Room DAO, сущности базы данных, JSON-модели, мапперы и реализации репозиториев;
- `worker` - фоновые задачи для сброса дневного прогресса и подготовки слов к повторению;
- `di` - Hilt-модули для внедрения зависимостей.

Основные экраны приложения:

- главный экран с навигацией;
- экран изучения новых слов;
- экран повторения;
- экран списка слов;
- экран статистики.

## Алгоритм повторения

При ответе пользователя приложение пересчитывает параметры повторения слова:

- если слово не вспомнили, количество повторений сбрасывается, а следующее повторение назначается через 2 часа;
- если слово вспомнили, интервал увеличивается по шагам: 2 часа, 8 часов, 1 день, 3 дня, затем масштабируется через ease factor;
- минимальный ease factor ограничен значением `1.3`.

Фоновая задача `RepeatWordsWorker` периодически переводит слова, у которых наступило время повторения, в состояние повторения. `ResetProgressWorker` обновляет дневной прогресс и стрик пользователя.

## Данные

Начальные данные находятся в каталоге:

```text
app/src/main/assets/
```

В проекте используются:

- `dictionary_data.json` - словарь;
- `tag_data.json` - теги слов.

При первом запуске данные загружаются в локальную Room-базу.

## Требования

- Android Studio;
- JDK 11;
- Android SDK с `compileSdk 35`;
- устройство или эмулятор с Android 7.0+ (`minSdk 24`).

## Запуск

1. Открыть проект в Android Studio.
2. Дождаться Gradle Sync.
3. Выбрать устройство или эмулятор.
4. Запустить конфигурацию `app`.

Также debug-сборку можно собрать из терминала:

```bash
./gradlew assembleDebug
```

## Проверки

Запуск unit-тестов:

```bash
./gradlew test
```

Запуск инструментальных тестов:

```bash
./gradlew connectedAndroidTest
```

Проверка стиля и статический анализ:

```bash
./gradlew ktlintCheck detekt
```

## Тестирование

В проекте есть unit-тесты для логики интервального повторения и инструментальные тесты для Android-части приложения. Основной пример бизнес-логики покрыт в `ProcessReviewAnswerUseCaseTest`: тесты проверяют успешное повторение, пересчет интервала и сброс прогресса при неправильном ответе.
